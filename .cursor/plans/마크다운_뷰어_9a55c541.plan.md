# 메모 편집 마크다운 뷰어 계획

## 라이브러리 선택

- 권장: `react-markdown` + `remark-gfm`
- 이유: Context7 문서 기준 `react-markdown`은 마크다운 문자열을 React 요소로 안전하게 렌더링하며, `components` 옵션으로 태그별 Tailwind 스타일 커스터마이징이 가능합니다. `remark-gfm`을 함께 쓰면 체크박스, 표, 취소선, URL 자동 링크 같은 GitHub Flavored Markdown을 지원할 수 있습니다.
- 보안 방침: `rehype-raw`는 사용하지 않습니다. 사용자가 입력한 raw HTML을 그대로 렌더링하면 XSS 위험이 생기므로, 메모 앱에서는 기본적으로 HTML을 escape/무시하는 `react-markdown` 기본 동작을 유지합니다.

## UX 설계

- [src/components/MemoForm.tsx](src/components/MemoForm.tsx)의 `내용` 입력 영역에 `작성` / `미리보기` 탭을 추가합니다.
- `작성` 탭에서는 현재 `textarea`를 그대로 사용합니다.
- `미리보기` 탭에서는 현재 `formData.content`를 마크다운으로 렌더링합니다.
- 내용이 비어 있으면 `미리보기할 내용이 없습니다.` 같은 안내 문구를 표시합니다.
- 저장 데이터 구조는 변경하지 않습니다. LocalStorage에는 기존처럼 `content: string`만 저장합니다.

## 구현 파일

- [package.json](package.json): `react-markdown`, `remark-gfm` 의존성 추가
- [src/components/MarkdownPreview.tsx](src/components/MarkdownPreview.tsx): 재사용 가능한 마크다운 뷰어 컴포넌트 생성
- [src/components/MemoForm.tsx](src/components/MemoForm.tsx): 작성/미리보기 탭 상태와 UI 연결
- 선택 사항: [src/components/MemoDetailModal.tsx](src/components/MemoDetailModal.tsx)의 상세보기 내용도 같은 `MarkdownPreview`로 렌더링하면 작성 화면과 상세보기 화면의 표시가 일관됩니다.

## 구현 방향

- `MarkdownPreview`는 클라이언트 컴포넌트로 만들고 `content: string` prop만 받습니다.
- `react-markdown`의 `remarkPlugins={[remarkGfm]}`를 사용합니다.
- `components` 매핑으로 기본 HTML 태그를 Tailwind 스타일에 맞춥니다. 예: `h1`, `h2`, `p`, `ul`, `ol`, `blockquote`, `code`, `pre`, `a`, `table`, `th`, `td`.
- 링크는 새 탭으로 열고 `rel="noopener noreferrer"`를 적용합니다.
- 코드블록 하이라이팅 라이브러리는 이번 범위에서는 추가하지 않습니다. 필요하면 나중에 `react-syntax-highlighter`나 Shiki 계열을 별도 검토합니다.

## 검증 계획

- `npm run lint`로 타입/린트 오류를 확인합니다.
- `npm run build`로 Next.js 15 빌드에서 클라이언트 컴포넌트와 의존성 문제가 없는지 확인합니다.
- 수동 확인 항목: 일반 문단, 제목, 굵게/기울임, 목록, 체크박스, 표, 링크, 인라인 코드, 코드블록, raw HTML 입력 시 안전한 렌더링 동작.
